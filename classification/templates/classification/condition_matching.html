Condition matching collects classifications that share the same "condition under curation", and allows you to assign standard ontology terms to them.
If a free text condition is provided, text will be matched to MONDO terms as a first pass (see https://mondo.monarchinitiative.org/ for more information)
e.g. The text "sifrim-hitz-weiss syndrome" can be assigned to MONDO:0014946

Levels
Standard terms can be assigned at the following levels:
  • Default for the condition text
  • For a specific gene symbol within that condition text
  • For a combination of mode of inheritance within the gene symbol
  • For a specific classification
At each level, if no value is set, it will inherit the values above it.
The goal is to have all classifications linked to a standard term. e.g. to end with Outstanding Classifications : 0
New classifications will automatically inherit values set here.

Edit
In many cases you'll need to edit a row to add terms.
It is recommended that terms be provided at the gene symbol level.
After clicking edit a popup will appear, with any chosen terms listed at the top (you can paste OMIM and MONDO terms here directly if needed).
You will see any known terms with gene level associations, and the top text matches.
You can change the text that's being searched and click "Search" for updated results.
Click toggle to add/remove a term, if multiple terms are selected you'll be required to choose "Co-occurring" or "uncertain"

Quick Suggestions
In some cases {{ site_name }} will be able to provide suggestions by looking at the text and gene symbols.
Review any warnings with the suggestions, untick any suggestions that are not appropriate, and hit "Approve" to bulk approve all the selected suggestions.
