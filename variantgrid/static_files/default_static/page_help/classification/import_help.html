The import page is to help power users understand how the import API works for creating automated uploads. It is not intended for general use.
You can use the form to provide an example record, and then see at the bottom left what JSON that maps to.
Clicking the Run button uploads the data to the server and shows the response.
If test mode and delete were off, after you Run a link to the newly created (or just updated) record will appear.

Test Mode The test mode checkbox means whatever you do will not have an effect on the database. Note that some of the return codes will be different when you do this, e.g. the id will be null instead of a valid value.

Lab Which lab is this record uploaded as, selection will be limited to the ones the current user is associated with.

Lab Record ID The id to be associated to this record, should be unique within the lab for new records, or the same as an existing record for an update.
It will default to a new ID on page load, but won't change automatically after that.

Delete If checked the delete flag will be sent up, should not be sent up on new record ids.

Publish Level What share level this data should be sent up as. Note this will only take effect if there are no validation errors.

c.hgvs & genome build The basic details for matching a coordinate. Note that the API does allow for variant coordinate instead of c.hgvs, but c.hgvs is preferred.

Clinical Significance, Zygosity Mandatory fields for a record - will only accept certain values.

key x, value x For providing values for any key found in the list of evidence keys.
e.g. gene_symbol  PTEN
All values will be sent up as strings, though the API will automatically convert strings to numbers, booleans and lists when appropriate.

Extra Data there is a text field for appending arbitrary JSON to the request.
This will allow you to upload extra bits of data such as notes and explain, e.g.

{
    "allele_origin": {
        "value":"germline",
        "note":"I think this is germline",
        "explain":"Our lab uses the following criteria to determine allele_origin"
    },
    "gnomad_af": 0.0002
}
